# 프로젝트 개요 (Project Overview)
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 개발자 및 크리에이터를 위한 맞춤형 멀티 링크 프로필 서비스 제공. 흩어져 있는 다양한 포트폴리오, 소셜 미디어, 작업물 링크를 하나의 페이지로 모아 쉽게 공유할 수 있도록 지원.
- **대상 사용자**: 개발자 및 크리에이터

---

# 핵심 기능 목록 (Key Features)

- **사용자 인증 (Auth)**: Firebase를 활용한 구글(Google) 소셜 로그인 기능
- **프로필 관리**: 구글 이메일 기반의 displayName, 간단한 소개글(Bio) 설정 및 수정 기능 (이미지 업로드 없음)
- **링크 관리 (CRUD)**:
  - 링크 추가, 삭제 및 **인라인 편집(Inline Edit)**을 통한 수정 기능
  - 링크 아이템 표시 시 구글 파비콘 API(Google Favicon API)를 활용해 해당 URL의 파비콘을 아이콘으로 적용
- **퍼블릭 프로필 페이지**: 별도의 핸들 기호(@) 없이 사용자의 **displayName**을 기반으로 한 URL (예: mylink.com/displayName) 제공
- **(추후 추가 예정)** 링크 클릭 조회수 통계 기능

---

# 각 기능의 상세 설명 (Feature Details)

### 1. 사용자 인증 (구글 소셜 로그인)
- **Firebase Authentication**을 연동하여 오직 구글 소셜 로그인만 지원합니다. 
- 복잡한 가입 절차를 생략하여 빠른 서비스 진입을 돕습니다.

### 2. 프로필 관리
- **자동 displayName 생성 및 수정**: 최초 가입 시 사용자의 구글 지메일(Gmail) 아이디 앞부분을 추출하여(예: abc@gmail.com -> abc) displayName의 기본값으로 자동 설정합니다. 사용자는 이 displayName과 짧은 텍스트 소개글(Bio)을 언제든 자유롭게 수정할 수 있습니다.
- **고유 URL(displayName) 적용**: 별도의 username 필드 없이, 설정된 displayName이 그대로 프로필 접속 주소가 됩니다.
  - **예약어 방어 로직**: 라우팅 충돌 방지를 위해 login, admin, settings 등 시스템에서 사용하는 **예약어(Reserved Words)는 displayName으로 사용할 수 없도록 제한**합니다.

### 3. 링크 관리
- **인라인 편집(Inline Edit)**: 링크의 타이틀과 URL을 수정할 때 별도의 페이지 이동이나 모달 창 없이, 리스트 텍스트를 클릭해 제자리에서 바로 수정할 수 있습니다.
- **자동 파비콘(Favicon) 적용**: 구글 파비콘 API를 호출하여 등록된 URL의 웹사이트 아이콘을 자동으로 가져와 링크 아이템 좌측에 표시합니다.

### 4. 퍼블릭 프로필 페이지 (사용자 노출 페이지)
- 사용자가 설정한 프로필 정보(displayName, 소개글)와 링크 목록이 최종적으로 렌더링되는 랜딩 페이지입니다. 
- 복잡한 테마 커스텀 기능 없이 기본적이고 깔끔한 UI로만 구성됩니다.

### 5. 데이터베이스 모델링 (Database Modeling)
- **Firebase Firestore**를 데이터베이스로 사용합니다.
- 사용자 계정 정보(User Document) 내에 고유 URL 및 화면 노출 용도로 모두 사용될 **displayName** 필드를 필수로 포함하여 설계합니다.
- 사용자가 등록하는 링크 목록 등의 데이터는 단일 문서 배열이 아닌 **서브 컬렉션(Sub-collection)** 구조를 사용하여 저장 및 관리의 효율성을 높입니다.
