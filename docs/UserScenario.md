# 사용자 시나리오 (User Scenarios)

## 1. 소유자 (Owner) 시나리오

### 1.1. 회원가입 및 로그인
- 사용자는 **서비스를 시작하기 위해** 구글(Google) 소셜 로그인 버튼을 클릭한다.
- 사용자는 **가입 직후 자신의 프로필 주소를 확보하기 위해** 지메일 아이디 앞부분으로 자동 할당된 `displayName`을 확인한다.

### 1.2. 프로필 관리
- 사용자는 **자신을 더 잘 표현하기 위해** 화면 최상단의 `displayName` 텍스트를 클릭하여 원하는 이름으로 인라인 편집(Inline Edit) 후 엔터(Enter)를 눌러 저장한다.
- 사용자는 **시스템 오류를 방지하기 위해** `admin`, `login` 등의 예약어를 `displayName`으로 변경 시도할 시 노출되는 에러 메시지를 확인하고 다른 이름으로 수정한다.
- 사용자는 **방문자에게 자신을 소개하기 위해** 소개글(Bio) 영역을 클릭하여 인라인 편집으로 내용을 작성하고 입력창 밖을 클릭(Blur)하여 저장한다.

### 1.3. 링크 관리 (추가/수정/삭제)
- 사용자는 **가입 직후 빈 화면을 채우기 위해** [새 링크 추가] 버튼을 클릭하여 첫 링크 영역을 생성한다.
- 사용자는 **방문자에게 공유할 목적지를 등록하기 위해** 생성된 빈 텍스트 입력창에 링크의 타이틀과 URL을 입력한다.
- 사용자는 **정상적인 링크 작동을 보장하기 위해** 잘못된 형태의 URL 입력 시 나타나는 경고 텍스트를 확인하고 올바른 주소 형태로 수정한다.
- 사용자는 **기존에 등록된 링크 정보를 변경하기 위해** 링크 아이템의 텍스트(타이틀 또는 URL)를 마우스로 클릭하여 제자리에서 인라인 편집을 수행한다.
- 사용자는 **링크의 성격을 시각적으로 알리기 위해** URL 등록 완료 즉시 구글 파비콘 API를 통해 자동으로 불러와진 해당 웹사이트의 아이콘(Favicon)을 확인한다.
- 사용자는 **더 이상 필요 없는 링크를 목록에서 제외하기 위해** 해당 링크의 [삭제] 아이콘을 누르고, 실수 방지용 확인 창(Confirm 모달)에서 '확인'을 클릭하여 최종 삭제한다.

### 1.4. 프로필 공유 및 통계 확인
- 사용자는 **자신의 멀티 프로필을 타인에게 홍보하기 위해** 화면에 배치된 [내 주소 복사] 버튼을 눌러 `mylink.com/displayName` URL을 클립보드에 복사한다.
- 사용자는 **자신의 링크들 중 어떤 것이 인기 있는지 파악하기 위해** 링크별로 기록된 '클릭 조회수(Count)'를 확인한다.

## 2. 방문자 (Visitor) 시나리오

### 2.1. 프로필 방문 및 링크 이동
- 방문자는 **소유자가 공유한 정보를 확인하기 위해** 소유자의 `mylink.com/displayName` 주소로 접속한다.
- 방문자는 **소유자의 정보를 빠르게 파악하기 위해** 별도의 복잡한 테마 없이 깔끔하게 렌더링된 이름, 소개글, 링크 목록(파비콘 포함)을 직관적으로 확인한다.
- 방문자는 **원하는 목적지로 이동하기 위해** 프로필 목록에서 특정 링크 아이템을 클릭한다.
- 방문자는 **이후에도 소유자의 마이링크 프로필 화면을 계속 탐색하기 위해** 클릭한 링크가 현재 창이 아닌 새 창(새 탭)으로 열리는 것을 경험한다.

### 2.2. 통계 데이터 생성 (백그라운드)
- 방문자는 **소유자에게 유의미한 클릭 데이터를 제공하기 위해** 특정 링크를 클릭하는 순간, 백그라운드에서 해당 링크의 '클릭 카운트'가 1 증가하도록 만든다. (페이지 조회수 등은 제외하고 순수 링크 클릭수만 집계)
